**졸업 프로젝트 중간 보고서: 언리얼 엔진 커스텀 셰이더 파이프라인 기반 물리 기반 Skinning 구현을 위한 사전 검증**

본 문서는 언리얼 엔진 5.5 환경에서 커스텀 셰이더 파이프라인을 통한 물리 기반(Physically-based) Skinning 시스템 구축을 최종 목표로 하는 졸업 프로젝트의 중간 진행 상황을 정리한 기술 문서입니다. 본 프로젝트의 핵심 과제는 머터리얼 그래프나 애니메이션 블루프린트 같은 고수준 추상화에 의존하지 않고 GPU 단계에서 직접 정점 변형(Skinning) 연산을 수행할 수 있는 자체 셰이더 파이프라인을 구축하는 것이며, 이를 위해서는 (1) 엔진 렌더링 파이프라인 사이에 사용자 패스를 안전하게 삽입하는 메커니즘, (2) 글로벌 셰이더와 HLSL 결합, (3) RDG(Render Dependency Graph) 기반 GPU 자원 관리, (4) 게임-렌더 스레드 간 데이터 브릿지 등 4가지 핵심 기술의 사전 검증이 선행되어야 합니다.

본 중간 단계에서는 위 4가지 기술 요소를 한꺼번에 검증할 수 있는 **단계형 시각 처리(Staged Rendering) 파이프라인**을 테스트베드로 구현하여, 향후 물리 기반 Skinning 셰이더로 확장 가능한지를 사전 검증하였습니다. 단계형 시각 처리는 Skinning에 비해 입출력이 단순하지만 동일한 6요소 부트스트랩 구조(모듈 부트스트랩 / SVE / 글로벌 셰이더 / RDG / HLSL / CVar 브릿지)를 필요로 하므로, 실제 Skinning 단계로 진입하기 전 파이프라인 골격을 안정화시키는 데 적합한 검증 대상입니다.

아래는 검증 과정에서 구현된 C++/HLSL 코드의 핵심인 `FStagedRenderSceneViewExtension::AfterTonemap_RT` 함수와 `StagedRender.usf`의 `MainPS` 진입점을 중심으로 한 상세 검증 내역입니다.

---

### 1. Scene View Extension 기반 커스텀 패스 삽입 검증 (`FVR_Study1Module`)

본 검증은 향후 물리 기반 Skinning 셰이더가 엔진 렌더링 파이프라인 사이에 안전하게 삽입될 수 있는지를 확인하는 것이 목적입니다. Skinning 셰이더는 정점/지오메트리 단계에 후크되어야 하지만, 이 단계에서는 가장 표준적이고 검증이 쉬운 PP(Post-Process) 단계 후크를 통해 SVE 기반 패스 삽입 메커니즘 자체를 우선 확인하였습니다.

* **모듈 종속성 및 셰이더 가상 디렉토리 매핑:** 글로벌 셰이더와 RDG, SVE를 사용하기 위해 `Build.cs`에 `RenderCore`/`RHI`/`Renderer`/`Projects` 4개 모듈을 종속성으로 추가하였습니다. 또한 `StartupModule` 단계에서 `AddShaderSourceDirectoryMapping` API를 호출하여 가상 경로 `/Project/VR_Study1`을 실제 폴더 `<ProjectDir>/Shaders`로 결합함으로써, `IMPLEMENT_GLOBAL_SHADER` 매크로가 USF 파일을 정확히 인식할 수 있도록 처리하였습니다. 이 매핑 구조는 추후 Skinning 셰이더(`Skinning.usf`)로 확장될 때 그대로 재사용 가능합니다.

* **`OnPostEngineInit`을 활용한 SVE 생성 지연:** `FSceneViewExtensions::NewExtension`은 내부적으로 `GEngine`을 요구하지만 `.uproject`의 `LoadingPhase`가 `PostConfigInit`이라 `StartupModule` 시점에는 `GEngine == nullptr`인 문제가 존재합니다. 이를 회피하기 위해 `FCoreDelegates::OnPostEngineInit`에 콜백을 등록하여 엔진 초기화 직후 SVE를 생성하도록 시점을 지연시켰으며, `TSharedPtr<..., ESPMode::ThreadSafe>`로 모듈 내 보유하여 셧다운 시 자동 해제되도록 처리합니다. 이 패턴은 모든 커스텀 SVE에 공통적으로 필요한 부트스트랩 절차이며 검증 완료되었습니다.

* **패스 후크 위치 검증:** `SubscribeToPostProcessingPass` 가상 함수를 오버라이드하여 `EPostProcessingPass::Tonemap` 단계에 `FAfterPassCallbackDelegate`를 등록하는 흐름을 검증하였습니다. 향후 Skinning 단계에서는 PP가 아닌 정점 처리 단계 후크가 필요하므로 다른 SVE 가상 함수(`PreRenderViewFamily_RenderThread` 등)를 사용해야 하지만, 콜백 등록과 RDG 빌더 접근 메커니즘은 본 검증을 통해 동작 패턴을 확인하였습니다.

### 2. 글로벌 셰이더 결합 및 RDG 디스패치 메커니즘 검증 (`AfterTonemap_RT` 함수)

본 검증은 향후 Skinning 셰이더가 GPU 자원(정점 버퍼, 본 매트릭스 SBO 등)을 안전하게 바인딩하고 디스패치되는 메커니즘을 확인하는 것이 목적입니다. 본 단계에서는 입력으로 텍스처를, 출력으로 RT를 사용하는 단순한 케이스로 RDG 그래프 등록과 자원 추적 동작을 검증하였습니다.

* **`IMPLEMENT_GLOBAL_SHADER` 결합 및 파라미터 구조체:** `FStagedRenderPS` 클래스는 `FGlobalShader`를 상속받으며 `SHADER_USE_PARAMETER_STRUCT` 매크로로 `FParameters`를 바인딩 구조체로 사용합니다. 파라미터 구조체에는 `SHADER_PARAMETER_RDG_TEXTURE`로 SceneColor SRV, `SHADER_PARAMETER_SAMPLER`로 정적 샘플러, uniform 상수, `RENDER_TARGET_BINDING_SLOTS()`로 RT 슬롯이 포함됩니다. 이 구조는 향후 Skinning 단계에서 `SHADER_PARAMETER_RDG_BUFFER_SRV`(본 매트릭스/가중치 버퍼)와 `SHADER_PARAMETER_RDG_BUFFER_UAV`(변형된 정점 출력)로 자연스럽게 확장될 수 있음을 확인하였습니다.

* **단계 분기 및 패스스루 단축 처리:** `CVarStageMode.GetValueOnRenderThread()`로 현재 모드를 읽은 후, 처리가 불필요한 케이스에서는 입력을 그대로 반환하여 셰이더 디스패치 자체를 생략(short-circuit)하는 패턴을 적용하였습니다. 입력 슬라이스가 무효한 예외 상황에서도 `Inputs.ReturnUntouchedSceneColorForPostProcessing(GraphBuilder)`로 파이프라인이 끊기지 않도록 폴백 처리를 두었으며, 이는 향후 Skinning 비활성 본/LOD에 대한 단축 처리에도 동일하게 활용할 수 있습니다.

* **트랜션트 RDG 자원 생성 및 디스패치 등록:** 입력의 `FRDGTextureDesc`를 복제하여 `Reset()` 후 필요한 플래그를 추가하고 `GraphBuilder.CreateTexture`로 트랜션트 출력 텍스처를 생성합니다. RT 슬롯에는 `ERenderTargetLoadAction::ENoAction`을 지정하여 풀스크린 덮어쓰기에서 불필요한 RT 로드 비용을 제거하였으며, `FPixelShaderUtils::AddFullscreenPass` 호출로 PSO 생성·뷰포트 설정·풀스크린 트라이앵글 그리기를 자동화하였습니다. RDG는 그래프 종료 시 트랜션트 자원을 자동 회수하고 SRV/RTV 배리어를 자동 삽입하므로, 향후 Skinning 셰이더에서 다수의 중간 버퍼를 사용하는 멀티 패스 구성에도 동일 메커니즘을 그대로 적용 가능함을 확인하였습니다. `RDG_EVENT_NAME` 마커는 RenderDoc/UE Insights에서 단계별 패스 식별을 가능하게 하여 디버깅 환경 또한 검증되었습니다.

### 3. HLSL 셰이더 결합 및 분기 처리 검증 (`StagedRender.usf`)

본 검증은 HLSL 코드가 C++ 측 파라미터 구조체와 정확히 결합되는지, 그리고 단일 셰이더 내에서 모드별 분기 처리가 동작하는지를 확인하는 것이 목적입니다. 향후 물리 기반 Skinning 셰이더는 LBS(Linear Blend Skinning) / DQS(Dual Quaternion Skinning) / 물리 시뮬레이션 결과 적용 등 여러 모드를 단일 셰이더로 통합 관리할 가능성이 있으므로, CVar 기반 분기 패턴이 사전 검증되어야 합니다.

* **`SV_POSITION` 기반 UV 재구성 및 파라미터 바인딩:** `FInstancedScreenVertexShaderVS`로부터 SV_POSITION만 받아 `SvPosition.xy * InputTexelSize` 식으로 UV를 직접 계산하는 패턴을 구현하였으며, 이는 C++의 `InputTexelSize` uniform 전달과 셰이더 측 바인딩이 정확히 일치함을 검증하였습니다. 동일한 결합 메커니즘은 Skinning 셰이더에서 본 인덱스/가중치 attribute 입력으로 확장됩니다.

* **CVar 기반 단일 셰이더 다중 모드 분기 검증:** 단일 USF 파일에서 `StageMode` uniform 값에 따라 4가지 처리 경로(고정 검정 출력 / 3×3 Sobel luma 에지 검출 / 50% 탈색 + 4단 포스터라이즈 / 패스스루)를 분기 수행하도록 구현하였습니다. `StageMode`는 모든 픽셀에 동일한 값이 전달되므로 GPU 와프 발산 없이 uniform branch로 처리됨을 확인하였으며, 향후 Skinning 모드 전환에도 동일 패턴이 적용 가능함을 검증하였습니다. 톤매핑 후 LDR 가정 하에서 `floor(*4)/4` 양자화와 `saturate(SobelEdge * 8.0)` 게인이 일관된 결과를 만드는 것을 시각적으로 확인하였습니다.

### 4. 게임 - 렌더 스레드 동기화 및 입력 트리거 검증 (`APCPawn::ApplyStage`)

본 검증은 향후 Skinning 시뮬레이션에서 게임 스레드의 시뮬레이션 상태(본 변환, 물리 파라미터)가 렌더 스레드의 셰이더 디스패치에 안전하게 전달되는 메커니즘을 확인하는 것이 목적입니다.

* **CVar 브릿지를 통한 스레드 경계 처리:** 별도의 `ENQUEUE_RENDER_COMMAND` 매크로 사용 없이 `IConsoleManager::Get().FindConsoleVariable(TEXT("stage.Mode"))->Set(...)` 호출 한 줄로 게임↔렌더 스레드 동기화가 가능함을 검증하였습니다. UE의 콘솔 변수는 내부적으로 게임/렌더 스레드 캐시를 분리 관리하므로, 게임 스레드에서 `Set` 후 다음 렌더 프레임에서 `GetValueOnRenderThread`로 안전하게 새 값이 전달됩니다. 본격 Skinning 단계에서는 다량의 본 변환 행렬을 전달해야 하므로 CVar 대신 `FRDGUploadData` 또는 Structured Buffer 업로드 패턴이 필요하지만, 본 검증을 통해 게임-렌더 동기화의 근본적 메커니즘은 확인하였습니다.

* **입력 매핑 및 모드 순환 트리거:** `Config/DefaultInput.ini`에 `PC_StageAdvance` 액션을 `One`(숫자 1) 키로 매핑하고, `SetupPlayerInputComponent`에서 `IE_Pressed` 이벤트를 핸들러에 바인딩하였습니다. 한 번 누를 때마다 4단계 순환 전환이 즉각 셰이더에 반영되는 것을 확인하였으며, `BlueprintCallable`로 노출하여 향후 시뮬레이션 시나리오에서 BP/시퀀서 트리거도 가능하도록 처리하였습니다.

* **테스트 검증 항목:** 1키 입력에 따른 4단계 순환 동작, 콘솔 명령 `stage.Mode N` 직접 입력 시 즉시 반영, 디스패치 단축 케이스에서의 비용 0 확인, RenderDoc에서 단계별 `RDG_EVENT_NAME` 마커 노출, `LogStagedRender`/`LogPCStage` 디버그 카테고리 및 `AddOnScreenDebugMessage`로 현재 단계 가시화 등 항목을 검증 완료하였습니다.

### 향후 개발 계획

현재 단계에서 언리얼 엔진 커스텀 셰이더 파이프라인 구축의 6요소 골격(모듈 부트스트랩, SVE 후크, 글로벌 셰이더 결합, RDG 자원 관리, HLSL 분기 처리, 게임-렌더 동기화)이 안정적으로 동작함을 사전 검증하였습니다. 다음 마일스톤에서는 이 골격을 본격적인 물리 기반 Skinning 파이프라인으로 확장할 예정입니다. 구체적으로 (1) PP 단계 후크를 정점 처리 단계 후크로 전환하여 USkeletalMeshComponent의 정점 스트림에 접근하고, (2) `FStagedRenderPS` 자리를 본 매트릭스 SBO와 가중치 버퍼를 입력으로 받는 컴퓨트 셰이더로 교체하며, (3) CVar 브릿지를 Structured Buffer 업로드 경로로 확장하여 다량의 본 변환을 매 프레임 안전하게 전달하도록 구현할 예정입니다. 또한 LBS와 DQS 알고리즘의 단일 셰이더 통합 관리, 강체-연체 결합 시뮬레이션 결과의 GPU 적용, RenderDoc 기반 정점 변형 검증 워크플로 정립 등을 병행하여 최종적으로 물리 기반 Skinning이 실시간 동작하는 환경을 구축하는 것을 목표로 합니다.
