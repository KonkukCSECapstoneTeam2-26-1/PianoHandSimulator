**졸업 프로젝트 중간 보고서: 2.5 IK 시스템 상세 구현 분석**

본 문서는 피아노 연주 시뮬레이션 및 고정밀 렌더링 시스템 구축을 위한 역운동학(Inverse Kinematics, IK) 모듈의 중간 진행 상황을 정리한 기술 문서입니다. 본 프로젝트의 IK 계산은 실시간이 아닌 오프라인 사전 계산 단계에서 수행되므로 수렴 속도보다 정확도와 제약 처리 유연성을 우선하여 설계되었습니다. 특히 특이점(Singularity) 문제를 회피하기 위해 DLS(Damped Least Squares) 방식이 적용된 Jacobian 기반 IK를 채택하여 구현을 진행하고 있습니다.

아래는 구현된 C++ 코드의 핵심인 `initHand` 함수와 `updateJacobianKinematics` 함수를 중심으로 한 상세 요구사항 충족 내역입니다.

---

### 1. 손 모델 계층 구조 및 생리적 ROM 한계 설정 (`initHand` 함수)

`initHand` 함수는 인체 해부학적 구조를 반영하여 손가락 모델의 기본 골격과 제약 조건을 초기화하는 역할을 담당합니다.

* **관절 계층 구조:** 시스템은 손목(Wrist) → 손바닥(Metacarpal) → MCP → PIP → DIP → 손가락 끝(Fingertip)으로 이어지는 관절 계층을 기반으로 합니다. 코드에서는 5개의 손가락(엄지 포함) 배열을 생성하고, 각 손가락마다 4개의 관절 위치(`localPos`)를 상대 좌표계로 할당하여 뼈대를 구축합니다.


* **관절별 ROM(Range of Motion) 제약 적용:** 부자연스러운 꺾임을 방지하기 위해 각 관절마다 엄격한 가동 범위를 적용합니다. 코드 내 `deg2rad` 람다 함수를 통해 각 관절의 제약 한계를 라디안으로 변환하여 `limits` 구조체에 저장합니다.
* **엄지 손가락 (인덱스 0):** 엄지 CMC 관절은 굴곡 0°~50°, 신전 0°~50°, 내외전 ±40°, 축회전 0°~15°의 가동 범위를 가지도록 설정됩니다. 또한 엄지 MCP는 굴곡 0°~60°, 신전 0°로 제한되며 , IP는 굴곡 0°~80°, 신전 0°~5° 한계를 갖도록 파라미터가 부여됩니다.


* **일반 손가락 (인덱스 1~4):** 나머지 손가락들의 MCP는 굴곡 0°~90°, 신전 0°~20°, 내외전 ±20° 범위로 설정됩니다. PIP는 굴곡 0°~100°, 신전 0°~10°로 제한되며 , DIP는 굴곡 0°~80°, 신전 0°~5°로 설정됩니다.




* **루트 오프셋 (Root Offset) 조정:** 건반을 치기 위한 자연스러운 손 모양을 잡기 위해 엄지는 Y축 아래 및 Z축 방향으로 시프트 시키고, 검지부터 새끼손가락은 부채꼴 모양으로 펴지도록 `rootOffset` 간격을 조정하였습니다.

### 2. Jacobian 기반 IK 연산 및 모션 생성 (`updateJacobianKinematics` 함수)

`updateJacobianKinematics` 함수는 설정된 타겟(피아노 건반)으로 손가락이 정확하게 도달할 수 있도록 IK 연산을 수행하고 궤적을 갱신합니다. 음표의 타깃 위치(건반 중심점)를 IK 목표로 설정하여 손가락 끝이 정확히 도달하도록 동작합니다.

* **DLS (Damped Least Squares) 알고리즘 적용:** * 타겟의 목표 위치와 현재 End-effector(손가락 끝) 사이의 오차 벡터(`error`)를 구한 후, 오차가 허용치(`tolerance`) 미만이 될 때까지 반복 연산(Iteration)을 수행합니다.
* 매 반복마다 Forward Kinematics(`computeFK`)를 통해 관절의 글로벌 변환 행렬을 갱신하고, 회전 축 벡터(X, Y, Z)와 End-effector까지의 거리 벡터를 외적(Cross Product)하여 $3 \times 9$ 크기의 Jacobian 행렬을 생성합니다.
* 특이점(Singularity) 근처에서 값이 발산하는 문제를 막기 위해 `damping` 값(현재 0.05 적용)을 추가한 $JJ^T + \lambda^2 I$ 형태의 역행렬을 계산하여 손가락이 안정적으로 목표로 향하도록 유도합니다.


* **ROM 제약 처리 로직 보완:** * 수렴 안정성을 위해 계산된 각도 변화량(`dTheta`)이 과도하게 클 경우 최대 스텝(`maxStep`)으로 길이를 제한합니다.
* 최종 갱신된 관절 각도는 `initHand`에서 설정한 `limits`를 참조하여 `std::clamp` 함수를 통해 강제로 자름으로써 생물학적 한계를 이탈하지 않도록 경계 처리를 진행합니다.




* **코드 블렌딩 및 베지어 곡선 기반 손목 워킹:**
* 실제 피아노 연주 모션을 시뮬레이션하기 위해, C코드 타겟 좌표와 F코드 타겟 좌표를 시간에 따른 사인 파형(`blend`)으로 보간합니다.
* 코드가 전환되는 동안 손목이 단순히 직선 이동하지 않고 자연스럽게 들렸다가 내려오도록 2차 베지어 곡선을 이용해 궤적(`globalWristPos`)을 생성하여 모션의 퀄리티를 향상시켰습니다.

### 향후 개발 계획

현재 C++ 코드 레벨에서 C코드와 F코드를 오가는 손가락의 Jacobian IK 알고리즘 검증이 완료되었습니다. 다음 마일스톤에서는 이 오프라인 계산 데이터를 언리얼 엔진 5의 Animation 시스템으로 전달하는 데이터 파이프라인을 구축하고, 타 IK 모듈과의 다중 손가락 충돌 방지 로직 결합을 테스트할 예정입니다.
